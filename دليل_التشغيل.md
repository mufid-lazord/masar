# دليل تشغيل ونشر تطبيق مَسَار

هذا الدليل يشرح كل شي بالترتيب وبأبسط طريقة ممكنة.

---

## أولاً: شو تم تعديله في المشروع؟

كان تسجيل الدخول **وهميًا** — أي بيانات تكتبها كانت تفوّتك مباشرة، وما كان في حساب ينحفظ فعليًا. تم إصلاح هذا بالكامل:

- **إنشاء حساب جديد** يحفظ الاسم والبريد والهاتف وكلمة المرور على الجهاز فعليًا.
- **تسجيل الدخول** صار يتحقق من البيانات: لازم البريد وكلمة المرور يطابقوا حسابًا مسجّلًا، وإلا تظهر رسالة خطأ.
- ما عاد في "دخول كضيف" — لازم يكون عندك حساب للدخول.
- كل شخص يدخل بحسابه هو، واسمه وبريده يظهران في صفحة "حسابي".
- "تسجيل الخروج" صار ينهي الجلسة فعليًا.
- إذا كنت مسجّلًا من قبل، التطبيق يفتح على الرئيسية مباشرة بدون إعادة تسجيل دخول.
- تم تصغير الترويسة (الـ Header) في الرئيسية وصفحة الحساب.

> ملاحظة: البيانات تنحفظ محليًا على الجهاز نفسه (عبر shared_preferences). يعني الحسابات تنحفظ لكل جهاز/متصفح على حدة. لو بدك قاعدة بيانات مركزية يدخل عليها الجميع من أي جهاز، هذا يحتاج خادم (مثل Firebase) — أخبرني وأساعدك فيه.

---

## ثانياً: تشغيل التطبيق على جهازك (الطريقة السهلة)

بدل ما تروح كل مرة للـ terminal وتكتب الأوامر:

1. افتح المشروع في **VS Code**.
2. افتح أي ملف داخل مجلد `lib`.
3. اضغط زر **Run** (مثلث التشغيل ▶️ فوق على اليمين) أو زر **F5** من الكيبورد.
4. اختر المتصفح (Chrome) أو المحاكي إذا ظهر خيار.

أول مرة فقط، إذا ما اشتغل، افتح الـ terminal مرة وحدة واكتب:

```
flutter create --platforms=web .
flutter pub get
```

بعدها زر التشغيل ▶️ يكفي كل مرة.

---

## ثالثاً: رفع المشروع على GitHub

1. أنشئ حسابًا على https://github.com إذا ما عندك.
2. اضغط **New repository**، سمّه مثلاً `masar`، واتركه **Public**.
3. لا تضع أي علامة على "Add README" (المشروع فيه واحد).
4. على جهازك، افتح الـ terminal داخل مجلد المشروع واكتب الأوامر التالية مرة واحدة (بدّل الرابط برابط مستودعك):

```
git init
git add .
git commit -m "أول نسخة من مسار"
git branch -M main
git remote add origin https://github.com/USERNAME/masar.git
git push -u origin main
```

> `USERNAME` = اسم حسابك على GitHub.

---

## رابعاً: تشغيل التطبيق "على GitHub" كرابط ويب (الأهم)

المشروع فيه ملف جاهز (`.github/workflows/deploy.yml`) يبني التطبيق وينشره تلقائيًا. بعد رفع الكود:

1. ادخل صفحة المستودع على GitHub.
2. اضغط **Settings** ← من القائمة اليسرى اختر **Pages**.
3. تحت **Build and deployment** ← في خانة **Source** اختر **GitHub Actions**.
4. روح لتبويب **Actions** فوق، ستجد عملية البناء تعمل (أو شغّلها يدويًا من "Run workflow").
5. انتظر دقيقتين حتى تكتمل (علامة ✔️ خضراء).
6. الرابط يظهر هكذا:

```
https://USERNAME.github.io/masar/
```

افتح هذا الرابط من أي متصفح أو هاتف — التطبيق يشتغل مباشرة بدون VS Code ولا terminal. كل مرة تعدّل وترفع (`git push`)، يتحدّث الرابط تلقائيًا.

---

## مشاكل شائعة وحلولها

- **صفحة بيضاء على الرابط:** تأكد أنك اخترت "GitHub Actions" في إعدادات Pages، وأن عملية Actions انتهت بعلامة خضراء.
- **فشل البناء في Actions:** افتح العملية الحمراء واقرأ آخر سطر؛ غالبًا يكون خطأ في الكود — انسخ الرسالة وأرسلها لي.
- **`git` غير معروف:** ثبّت Git من https://git-scm.com ثم أعد فتح الـ terminal.
